On passe en mode Architecte de Données.Dans Convex, la structure de ta base de données doit être flexible car les besoins d'un visa pour les USA (très détaillé) ne sont pas les mêmes que pour Dubaï (très simple). On va utiliser un schéma avec des objets optionnels pour gérer cette diversité.Voici la structure logique de ta table principale applications :1. La Table applications (Le cœur du business)Chaque ligne de cette table représente un client et son dossier de visa.ChampTypeDescriptionuserIdstring (id Clerk)Lie le dossier au compte du client.type"USA" | "DUBAI" | "TURKEY" | "INDIA"La destination choisie.statusstringDRAFT, PAID_ENGAGEMENT, SEARCHING_SLOT, SLOT_FOUND, COMPLETED.personalInfoobjectNom, Postnom, Passeport, Date de naissance, Téléphone.documentsarrayListe d'IDs (Storage Convex) pour les scans (Passeport, Photo, Reçus).specificDataobject (flexible)Champs spécifiques (ex: Numéro SEVIS pour étudiant, Nom Hôpital pour médical).appointmentobject (optional)Date, Heure, Lieu et Numéro de confirmation une fois capturé.paymentStatusobjectengagementPaid: boolean, successFeePaid: boolean.2. Pourquoi cette structure est "Vibe Coding" compatible ?Évolutivité : Si demain tu veux ajouter le Canada ou la France, tu n'as pas besoin de recréer une base de données. Tu ajoutes juste une nouvelle valeur dans type et de nouveaux champs dans specificData.Sécurité : En liant le userId de Clerk, un client de Limete ne pourra jamais voir les documents d'un client de la Gombe.Filtrage Admin : Dans ton dashboard, tu pourras faire des requêtes ultra-rapides du genre : "Montre-moi tous les dossiers USA qui ont payé l'engagement mais qui n'ont pas encore de rendez-vous."3. Les Dossiers (Storage)On ne stocke pas les images (Passeport/Photos) directement dans la table.On utilise le Convex File Storage.La table contient juste le "lien" (fileId).C’est ce qui rend ton application ultra rapide, même avec des centaines de dossiers.4. La logique des Étapes (Workflow)Ton système va fonctionner par "Etapes Validées" :Étape 1 : Remplissage des infos de base.Étape 2 : Upload des documents (Passeport + Photo).Étape 3 : Paiement des frais d'engagement (Validation par toi).Étape 4 : Activation du script de "Slot Hunting" (Automatique).Étape 5 : Capture et notification (Le moment de victoire).
