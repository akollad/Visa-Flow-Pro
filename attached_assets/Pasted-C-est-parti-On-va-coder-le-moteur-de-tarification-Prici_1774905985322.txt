C'est parti. On va coder le moteur de tarification (Pricing Engine) dans Convex. L'idée est de centraliser toute la logique de prix pour que, si tu décides de changer un tarif un matin, cela se répercute instantanément sur tout le site.

Voici comment structurer tes fonctions Convex pour gérer les frais d'engagement et les primes de succès.

1. Définition des constantes de prix
On crée un fichier de configuration pour que ce soit facile à modifier sans toucher au code logique.

TypeScript
// convex/constants.ts

    label: "USA (B1/B2, F1, K1)",

export type Destination = keyof typeof VISA_PRICING;
2. La Mutation pour créer une demande
Cette fonction enregistre la demande initiale dans Convex et calcule automatiquement ce que le client doit payer.

TypeScript
// convex/applications.ts
import { mutation, query } from "./_generated/server";
import { v } from "convex/values";
import { VISA_PRICING, Destination } from "./constants";

export const createApplication = mutation({
  args: {
    destination: v.string(), // On valide que c'est bien une destination connue
    visaType: v.string(),
  },
  handler: async (ctx, args) => {
    const identity = await ctx.auth.getUserIdentity();
    if (!identity) throw new Error("Non autorisé");

    const destKey = args.destination as Destination;
    const pricing = VISA_PRICING[destKey];

    if (!pricing) throw new Error("Destination non supportée");

    const applicationId = await ctx.db.insert("applications", {
      userId: identity.subject, // ID de Clerk
      destination: args.destination,
      visaType: args.visaType,
      status: "AWAITING_ENGAGEMENT_PAYMENT",
      priceDetails: {
        total: pricing.total,
        engagementFee: pricing.engagementFee,
        successFee: pricing.successFee,
        paid: 0,
        isSuccessFeePaid: false,
      },
      logs: [{
        msg: `Dossier créé pour ${pricing.label}`,
        time: new Date().toISOString()
      }],
    });

    return applicationId;
  },
});
3. La logique Temps Réel (Le "Magic Moment")
C'est ici que Convex brille. Quand tu trouves un rendez-vous (en tant qu'Admin), tu déclenches cette mutation. L'écran du client va changer instantanément.

TypeScript
// convex/admin.ts
import { mutation } from "./_generated/server";
import { v } from "convex/values";

export const markSlotFound = mutation({
  args: { applicationId: v.id("applications") },
  handler: async (ctx, args) => {
    const app = await ctx.db.get(args.applicationId);
    if (!app) throw new Error("Dossier introuvable");

    // On met à jour le statut
    await ctx.db.patch(args.applicationId, {
      status: "SLOT_FOUND_AWAITING_SUCCESS_FEE",
      logs: [
        ...app.logs,
        {
          msg: "🎉 Bonne nouvelle ! Un rendez-vous a été capturé. Veuillez régler la prime de succès pour voir les détails.",
          time: new Date().toISOString()
        }
      ]
    });
    
    // Ici, tu pourrais aussi déclencher un email automatique via Resend
  },
});
4. Pourquoi ce code est "Smart" pour ton business ?
Flexibilité : Si tu veux lancer un "Promo Month" sur Dubaï, tu changes juste le chiffre dans constants.ts et tout le site (UI, factures, base de données) s'ajuste.

Sécurité : Les prix sont calculés côté serveur (Backend). Un petit malin ne peut pas modifier le prix dans son navigateur pour payer moins cher.

Expérience Client : Avec le système de logs, le client voit l'historique de ses paiements et de ses étapes directement sur son dashboard.
